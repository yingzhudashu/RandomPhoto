# 🔐 权限说明

本文档详细说明了随机相册应用所需的权限及其用途。

## 📱 所需权限

### 1. 相册/存储权限

**权限名称**: `READ_MEDIA_IMAGES` (Android 13+) / `READ_EXTERNAL_STORAGE` (Android 12 及以下)

**用途**:
- 读取相册中的照片用于随机展示
- 仅在应用运行时访问
- 不会后台扫描或上传照片

**为什么需要**:
- 应用的核心功能是从相册中随机选择照片
- 没有此权限无法访问照片

**安全说明**:
- ✅ 仅本地读取，不上传
- ✅ 仅在应用前台运行时访问
- ✅ 不收集任何个人信息
- ✅ 不分享给第三方

### 2. 删除照片权限

**权限名称**: `DELETE_PACKAGES` (通过系统删除 API)

**用途**:
- 删除用户确认的照片
- 需要用户二次确认

**安全说明**:
- ✅ 每次删除都需要用户确认
- ✅ 不会批量删除
- ✅ 不会删除系统照片

## 🛡️ 隐私保护

### 我们承诺

1. **最小权限原则**
   - 只申请必要的权限
   - 不过度索权

2. **透明使用**
   - 权限用途清晰说明
   - 开源代码可审查

3. **本地处理**
   - 所有数据本地存储
   - 不联网，不上传

4. **用户控制**
   - 随时可在系统设置中撤销权限
   - 删除操作需二次确认

### 权限管理

**查看权限**:
```
设置 → 应用 → 随机相册 → 权限
```

**撤销权限**:
```
设置 → 应用 → 随机相册 → 权限 → 关闭对应权限
```

**注意**: 撤销相册权限后，应用将无法显示照片。

## ❓ 常见问题

### Q: 为什么要相册权限？

**A**: 应用的核心功能是从相册随机展示照片，没有此权限无法实现基本功能。

### Q: 应用会上传我的照片吗？

**A**: **绝对不会！** 应用完全离线运行，不联网，不上传任何数据。

### Q: 权限安全吗？

**A**: 非常安全：
- 代码开源可审查
- 仅申请必要权限
- 本地处理不上传
- 用户完全控制

### Q: 可以只给部分照片权限吗？

**A**: Android 13+ 支持选择性相册权限：
- 系统会询问"允许访问所有照片"或"仅选择部分照片"
- 您可以选择只授权部分照片
- 应用会尊重您的选择

### Q: 撤销权限后数据会丢失吗？

**A**: 不会：
- 应用不存储任何数据
- 撤销权限只是禁止访问
- 重新授权后可继续使用
